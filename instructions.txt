Olá, esse arquivo tem por função explanar sobre a estrutura desse projeto, bem como os procedimentos para executa-lo em sua máquina.

Esse projeto web foi desenvolvido usando python nas versão 3.11.8 64 bits como linguagem padrão, e framework flask versão 3.0.2 e banco de dados foi usado o mysql, portanto para
executar esse sistema você que ter em sua máquina o python versão 3.11.8 + flask + 3.0.2 + mysql instalados, juntamente das bibliotecas de gerenciamento do banco de dados,
no nosso caso a pymysql, e ainda jsonify, request, MethodView ambos do flask, outro ponto que podemos destacar, e que esse sistema foi desenvolvido em um ambiente virtual - venv-venv;

Para instalação do python, você deve abrir o navegdor web e procurar por 'python development', entrarás na pag oficial do python e baixar a versão corrente mais recente,
apos isso, basta executar o .exe do python e ele vai se instalar, para verificar se tudo deu certo, abra o CMD, e digite "python --version", que ele irá mostrar versão do python instalada.

Com pytho instalado, teremos acesso ao gerenciador pip trabalhando por trás dele, por meio do terminal, poderemos realizar a instalação dos outros modulos, no caso do flask, basta digitar "pip install flask"
no terminal, o proprio terminal ira instalar a versão mais recente pra você.

Com isso, vamos a instalação do pymysql, bastando digitar "pip install pymysql" no terminal, a versão mais recente será baixada e instalada.

A estrutura desse sitema se baseia no modelo MVC "model view controller", e aplicamos ainda a estrutura de POO "Programação Orientada a Objetos", para que pudessemos deixar o projeto mais estruturado e organizado;

A estrutura do banco de dados se dá seguinte forma, primeiro, foir feita a conexão com banco de dados, passando para o arquivo "conection_db.py" as instruções e dados necessários para a conexão efetiva com banco,
posteriormente criamos um arquivo tipo .sql chamdo de 'db.sql', contendo os codigos em sql para criação e o start de uso do banco de dados, posteriormente a criação da tabela que vai receber os dados de cadastros 
dos produtos, contendo os tipos de cada campo, definição do campo com chave primária-'primary key', sendo esse o ID, que se incrementa a cada produto cadastrado, e os campos restantes com limitações de quantidade
de caracteres;

Espero que gostem do meu projeto ":)".
